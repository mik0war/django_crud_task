### **Техническое задание: Разработка CRUD-приложения «Библиотека» на Django**

**1. Цель проекта:**
Разработать веб-приложение для учета книг в библиотеке. Приложение должно предоставлять интерфейс для администратора (библиотекаря) для управления книгами через HTML-страницы, а также REST API для управления авторами.

**2. Требования к базе данных:**

**2.1. Используемая СУБД:**
- **PostgreSQL** - основная база данных приложения
- **Подключение PostgreSQL:**
  ```python
  # settings.py
  DATABASES = {
      'default': {
          'ENGINE': 'django.db.backends.postgresql',
          'NAME': 'library_db',
          'USER': 'your_username',
          'PASSWORD': 'your_password',
          'HOST': 'localhost',
          'PORT': '5432',
      }
  }
  ```
- **Предварительные шаги:**
  1. Установить PostgreSQL на сервер
  2. Создать базу данных `library_db`
  3. Установить пакет: `pip install psycopg2-binary`
  4. Выполнить миграции: `python manage.py migrate`

**2.2. Структура таблиц:**

*   **Таблица `Author`:**
    *   `id` (Primary Key)
    *   `first_name` (CharField) — Имя автора.
    *   `last_name` (CharField) — Фамилия автора.
    *   `bio` (TextField) — Биография автора (может быть пустым).
    *   `birth_date` (DateField) — Дата рождения (может быть пустым).

*   **Таблица `Genre`:**
    *   `id` (Primary Key)
    *   `name` (CharField) — Название жанра (уникальное).
    *   `description` (TextField) — Описание жанра (может быть пустым).

*   **Таблица `Book`:**
    *   `id` (Primary Key)
    *   `title` (CharField) — Название книги.
    *   `author` (ForeignKey to `Author`) — Ссылка на автора. При удалении автора, у книг это поле должно устанавливаться в `NULL`.
    *   `isbn` (CharField) — ISBN номер (уникальный).
    *   `publication_year` (IntegerField) — Год публикации.
    *   `genres` (CharField) — строка с перечислением жанров через запятую.
    *   `co_authors` (CharField) — строка с перечислением соавторов
    *   `summary` (TextField) — Краткое описание книги (может быть пустым).

**3. Функциональные требования (CRUD):**

**3.1. Функционал для сущности "Книга" (Book)**
Эти endpoints должны возвращать готовый HTML для отображения в браузере.

*   **Маршруты:**
    *   `GET /books/` — Показать страницу со списком всех книг.
    *   `GET /books/<int:book_id>/` — Показать страницу с детальной информацией о книге.
    *   `GET /books/new/` — Показать форму для добавления новой книги.
    *   `POST /books/new/` — Обработать данные формы и создать новую книгу.
    *   `POST /books/<int:book_id>/` — Обработать данные формы и обновить существующую книгу.
    *   `DELETE /books/<int:book_id>/` — Удалить книгу.

**3.2. Функционал для сущности "Автор" (Author)**
Эти endpoints должны принимать и возвращать данные в формате JSON.

*   **Маршруты:**
    *   `GET /authors/` — Получить JSON-массив всех авторов.
    *   `GET /authors/<int:author_id>/` — Получить JSON-объект с данными автора.
    *   `POST /authors/` — Создать нового автора. Принять JSON, вернуть созданный объект.
    *   `PUT /authors/<int:author_id>/` — Полностью обновить данные автора.
    *   `DELETE /authors/<int:author_id>/` — Удалить автора.

**4. Дополнительные требования:**
- Код должен быть четко структурирован
- Использовать Django Class-Based Views (CBV) там, где это уместно
- Приложение должно иметь логическую структуру

**5. Критерии оценки:**
1. Для получения оценки `отлично` необходимо реализовать полный функционал приложения, код должен иметь четкую и лаконичную структуру, таблицы в БД полностью соответствовать требованиям 3НФ
2. Для получения оценки `хорошо` необходимо реализовать функционал приложения, частично не соответствующий требованиям технического задания, код должен иметь четкую и лаконичную структуру
3. Для получения оценки `удовлетворительно` необходимо реализовать частичный функционал приложения - одно из приложений, код должен иметь четкую и лаконичную структуру
